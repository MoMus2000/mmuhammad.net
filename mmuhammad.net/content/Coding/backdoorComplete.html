How backdoors work on computers ...
By Mustafa Muhammad
In this article I will discuss backdoors, especially from the perspective of web developers who are well experienced with the server client architecture.

I would like to stress that the following article is for learning only and I do not accept any liability if the reader uses the gained knowledge for malicious purposes.

Let us first discuss what a backdoor is:
A backdoor is a program that allows an unauthorized user to gain control of your system with out the user or system being aware of it. Usually backdoors are utilized for malicious perposes hence are also commonly referred to as a trojan virus.

Essentially its shell/terminal access gained by a user operating from a Command and Control (C&C) server.
These backdoors are typically found bundled into software that from a bird's eye view look pretty safe out of the box.
Most of us developers rely on open source code to cut down on development time and prevent reinventing the wheel from scratch, however there is a level of trust involved when using these libraries of code found on the internet.
What makes it scary is if the libraries that we usually trust to be safe conatiner a backdoor that would be added into our code upon building/compiling the project.
A pretty well known example of this is the ken thompson hack, it talks about adding in backdoor into a C compiler, this would mean that every compiled C binary would contain a backdoor to be able to accessed by the attacker/unauthorized user.
How does a backdoor work?
A graphic that explains the architecture involved in a backdoor
So how would one write this in code ?
  1. Create a slave server and client server
  2. The client server would be programmed to listen to incoming connections
  3. The slave server would try and connect to the the client (C&C) machine
  4. Once the connection is establised, the possibilities are only limited to the what was programmed onto the slave server before being planted in the victim PC.


Why does this work?
While firewalls typically block unrequested incoming traffic, in this particular case the slave server is the one that initiates the request hence the request is conidered to be legitimate and is allowed to go through.
Programming languages like python, Go (Golang) and many others provide the ability to run shell commands via their standard library, hence these are mostly used to control the victim PC
While the steps in creating a backdoor seem easy, the hard part is planting the malicous code, hence this is what is meant by hacking consisting a large part of social engineering, rather than technical ability
In conclusion here are some helpful tips to avoid falling victim to a trojan virus:
  1. Vetting the files that you download from the internet.
  2. Monitor your running processes to see if there's anything unexpected, the same goes for checking the network to see some anomalous requests originating from your computer.
